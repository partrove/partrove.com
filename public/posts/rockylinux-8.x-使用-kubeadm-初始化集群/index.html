安装容器运行时#

containerd#

下载#

v1.6.25

解压安装#

注意对应的包名和路径。

mkdir /tmp/containerd && tar xvf /tmp/cri-containerd-1.6.25-linux-amd64.tar.gz -C /tmp/containerd && cp -r /tmp/containerd/* /

配置crictl#

cat << EOF | tee /etc/crictl.yaml
runtime-endpoint: unix:///run/containerd/containerd.sock
image-endpoint: unix:///run/containerd/containerd.sock
timeout: 10
debug: false
EOF

配置containerd#

首先生成默认配置，

mkdir /etc/containerd && containerd config default > /etc/containerd/config.toml

然后修改配置，配置 cgroup 驱动为 systemd，

sed -i 's#SystemdCgroup = false#SystemdCgroup = true#' /etc/containerd/config.toml

然后将默认的 pause 镜像版本，修改为跟所选的 k8s 版本相匹配，这里选择的 k8s 版本为 v1.26.8，对应的 pause 镜像版本为 3.9，

sed -i 's#registry.k8s.io/pause:3.6#registry.k8s.io/pause:3.9#' /etc/containerd/config.toml

启用containerd服务#

systemctl daemon-reload && systemctl enable containerd --now

安装 k8s 工具组件#

包管理方式安装#

cat <<EOF | tee /etc/yum.repos.d/kubernetes.repo
[kubernetes]
name=Kubernetes
baseurl=https://packages.cloud.google.com/yum/repos/kubernetes-el7-\$basearch
enabled=1
gpgcheck=1
gpgkey=https://packages.cloud.google.com/yum/doc/rpm-package-key.gpg
exclude=kubelet kubeadm kubectl
EOF

再安装指定版本 k8s 组件，

配置kubelet服务#

创建 kubelet.service 文件，

cat << EOF >> /etc/systemd/system/kubelet.service
[Unit]
Description=kubelet: The Kubernetes Node Agent
Documentation=https://kubernetes.io/docs/home/
Wants=network-online.target
After=network-online.target

[Service]
ExecStart=/usr/local/bin/kubelet
Restart=always
StartLimitInterval=0
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

创建 kubelet conf 文件，

mkdir -p /etc/systemd/system/kubelet.service.d && \
cat << EOF >> /etc/systemd/system/kubelet.service.d/10-kubeadm.conf
# Note: This dropin only works with kubeadm and kubelet v1.11+
[Service]
Environment="KUBELET_KUBECONFIG_ARGS=--bootstrap-kubeconfig=/etc/kubernetes/bootstrap-kubelet.conf --kubeconfig=/etc/kubernetes/kubelet.conf"
Environment="KUBELET_CONFIG_ARGS=--config=/var/lib/kubelet/config.yaml"
# This is a file that "kubeadm init" and "kubeadm join" generates at runtime, populating the KUBELET_KUBEADM_ARGS variable dynamically
EnvironmentFile=-/var/lib/kubelet/kubeadm-flags.env
# This is a file that the user can use for overrides of the kubelet args as a last resort. Preferably, the user should use
# the .NodeRegistration.KubeletExtraArgs object in the configuration files instead. KUBELET_EXTRA_ARGS should be sourced from this file.
EnvironmentFile=-/etc/default/kubelet
ExecStart=
ExecStart=/usr/local/bin/kubelet \$KUBELET_KUBECONFIG_ARGS \$KUBELET_CONFIG_ARGS \$KUBELET_KUBEADM_ARGS \$KUBELET_EXTRA_ARGS
EOF

启用 kubelet 服务#

systemctl daemon-reload && systemctl enable kubelet --now

初始化集群#

kubeadm init --cri-socket unix:///run/containerd/containerd.sock --kubernetes-version 1.28.2 --upload-certs --dry-run

  mkdir -p $HOME/.kube
  cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
  chown $(id -u):$(id -g) $HOME/.kube/config
